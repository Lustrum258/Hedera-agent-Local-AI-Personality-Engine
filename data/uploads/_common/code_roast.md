# Code Roast 🔥 — 代码审查机器人，嘴臭但专业

一个用 Python 写的代码静态审查工具。不跟你客气，有问题直接喷。

---

## 快速开始

```bash
# 审查一个文件
python code_roast.py your_code.py

# 自虐模式 —— 审查自己
python code_roast.py --self
```

## 它检查什么

| 检测项 | 说明 | 扣分 |
|--------|------|------|
| **行长** | 超过 100 个字符的行 | -2 |
| **尾随空格** | 行尾有多余空格 | -1 |
| **文件末尾换行** | 最后一行没有换行符 | -1 |
| **import 顺序** | 第三方库 import 放在标准库前面 | -3 |
| **注释质量** | 注释太短（<10字符）或含 TODO/FIXME | -1~-2 |
| **函数命名** | 函数名不是小写开头，或名字超过 30 字符 | -2 |
| **空行泛滥** | 连续空行超过 3 行 | -2 |
| **魔法数字** | 代码中直接出现裸数字（2, 3, 8, 60...） | -1 |
| **AST 空函数** | 函数体只有 docstring 没有实际代码 | -3 |
| **语法错误** | 代码无法通过 Python 解析 | -10 |

## 评分等级

| 分数 | 评语 |
|------|------|
| ≥ 90 | 不错，但别骄傲 |
| ≥ 70 | 能跑，但也就这样了 |
| ≥ 50 | 建议重写，趁还来得及 |
| < 50 | 删了吧，就当没写过 |

## 代码结构

```
code_roast.py
├── CodeRoast 类
│   ├── __init__()        — 初始化文件名、行列表、问题列表、分数
│   ├── load()            — 读取文件
│   ├── check_line_length() — 行长检查
│   ├── check_trailing_whitespace() — 尾随空格检查
│   ├── check_missing_newline() — 文件末尾换行检查
│   ├── check_import_order() — import 顺序检查
│   ├── check_comment_quality() — 注释质量检查
│   ├── check_function_naming() — 函数命名检查
│   ├── check_too_many_blank_lines() — 空行泛滥检查
│   ├── check_magic_numbers() — 魔法数字检查
│   ├── run_ast_check()   — AST 语法树检查
│   └── roast()           — 汇总输出
└── main()                — 入口
```

## 依赖

零依赖。只用 Python 标准库：`sys`, `re`, `os`, `ast`。

## 已知问题

- 魔法数字检测太粗暴，`range(10)` 里的 `10`、`sleep(0.5)` 里的 `0.5` 都会被报
- 注释质量检测只数长度，不判断内容是否有意义
- 不支持 TypeScript / Java 等其他语言 —— 暂时只喷 Python

---

> 写这个工具不是为了恶心你 —— 好吧，也有一点。
> 但主要是为了让你的代码不那么烂。
